# 2026-03-28 — Crate rename: lib/app/daemon naming convention

## What

Renamed all 24 crates to follow a naming convention that makes crate type visible from the name alone:

- `cosmix-lib-*` — libraries (Rust module name preserved as `cosmix_*` via `[lib] name`)
- `cosmix-*` — GUI apps (Dioxus desktop/web)
- `cosmix-*d` — headless daemons/services

## Rename map

| Old | New | Reason |
|-----|-----|--------|
| cosmix-port | cosmix-lib-amp | lib + name matches the AMP protocol |
| cosmix-client | cosmix-lib-client | lib |
| cosmix-config | cosmix-lib-config | lib |
| cosmix-mesh | cosmix-lib-mesh | lib |
| cosmix-ui | cosmix-lib-ui | lib |
| cosmix-jmap | cosmix-maild | daemon + name matches the role, not the protocol |
| cosmix-hub | cosmix-hubd | daemon |
| cosmix-embed | cosmix-indexd | daemon + name matches the intent (indexing), not the mechanism (embeddings) |
| cosmix-web | cosmix-webd | daemon |
| cosmix-launcher | cosmix-menu | app + stripped of Lua script management |
| (new) | cosmix-scripts | Lua + Bash script manager, extracted from cosmix-launcher |

## Key decisions

- **Flat directory structure retained** — naming convention provides visual grouping without subdirectories. Can move to `apps/`, `libs/`, `daemons/` folders later if the workspace grows past ~100 crates.
- **Rust module names preserved** for all libs except cosmix-port → cosmix_amp. This means zero source changes for the 4 libs that kept their module name. Only 5 .rs files needed `use cosmix_port` → `use cosmix_amp`.
- **Cargo.toml `package` key** used for lib deps: `cosmix_ui = { package = "cosmix-lib-ui", path = "..." }` — keeps `use cosmix_ui::` working while the crate name on disk is `cosmix-lib-ui`.
- **cosmix-launcher split** into cosmix-menu (pure tray app launcher) + cosmix-scripts (CLI script manager with bash support). Scripts dir moved from `~/.local/lua/` to `~/.local/scripts/`.
- **`_doc/` and `_journal/` left as historical record** — they document what was true when written.
