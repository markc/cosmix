# 2026-03-08 — ARexx lives: Phase 1 port discovery and routing, menu bars, libcosmic migration

## MILESTONE: Phase 1 — Port Discovery and Routing (ARexx Proof of Life)

**The ARexx model works on modern Linux.** External scripts and CLI commands now
control running COSMIC GUI apps in real-time via Unix socket IPC, with the
calculator's display updating live. This is the foundational moment — the first
working demonstration that the Amiga's universal inter-application scripting bus
can be rebuilt for the COSMIC desktop.

No modern Linux desktop has this. KDE has KWin scripting (window management
only). GNOME has D-Bus extensions (introspection-based, no unified scripting).
Neither provides a universal command bus where any app can be scripted by any
other. Cosmix is the first.

### What was built

**Port Registry** (`daemon/registry.rs`):
- `PortRegistry` and `PortInfo` structs in daemon shared state
- Directory scanner watches `/run/user/$UID/cosmix/ports/` for `.sock` files (2s poll)
- HELP handshake: on discovery, daemon connects and sends HELP to learn commands
- Heartbeat loop: dead ports cleaned up every ~10s
- Case-insensitive lookup with instance suffix support (COSMIX-CALC.1)

**Message Routing** (`lua/ports.rs` rewrite):
- `cosmix.port("cosmix-calc")` resolves to app socket via registry (or direct path fallback)
- App port tables have `:send(cmd, args)` method that calls the real Unix socket
- Built-in ports (clipboard, windows, etc.) still handled internally
- `json_to_lua()` converter for round-tripping JSON responses to Lua tables

**New Lua APIs**:
- `cosmix.port_exists(name)` — check if a port is registered
- `cosmix.list_ports()` — list all ports (built-in + app) with metadata
- `cosmix.wait_for_port(name, timeout_ms)` — block until port appears

**CLI commands**:
- `cosmix call <port> <cmd> [json]` — call any app port from the command line
- `cosmix list-ports` — show all registered ports

**IPC protocol** additions:
- `CallPort` — daemon routes to app socket from registry
- `ListPorts` — returns built-in + discovered app ports

**GUI live sync** (cosmix-calc):
- `AtomicBool` dirty flag set by port command handlers
- 100ms subscription polls dirty flag, triggers cache refresh from engine
- `calc` command now updates `expression` + `display` (was silent eval before)
- All mutating port commands (`calc`, `clear`, `memory`, `press`) set dirty flag

### The proof

```
$ cosmix-calc &                              # launch calculator
$ cosmix call cosmix-calc calc '"2+3*8"'     # send expression
"26"                                          # CLI gets result
# Calculator GUI display updates to "26" in real-time
```

```lua
-- Lua script orchestrating a running app
local calc = cosmix.port("cosmix-calc")
local r = calc:send("calc", "1250 * 1.1")
print(r.data)  --> 1375 (and GUI updates live)
```

This is Phase 1 of the 8-phase ARexx adoption plan. The semantic layer
(standard vocabulary, macro menus, clip list, orchestration) comes next.

## Native Wayland screenshot capture

Implemented full compositor screenshot using `ext-image-copy-capture-v1` protocol
in `wayland/screenshot.rs`. No external tools (grim, cosmic-screenshot, slurp) —
pure Rust/Wayland protocol. Captures all windows, overlays, and layers at native
resolution (3840x2160 on test system).

**Three interfaces:**
- **CLI**: `cosmix screenshot [path]` / `cosmix ss [path]`
- **IPC**: `Screenshot { path }` request via daemon
- **Lua**: `cosmix.screenshot(path?)` returns saved path

**Implementation details:**
- Binds `ext_image_copy_capture_manager_v1` and `ext_output_image_capture_source_manager_v1`
- Creates SHM buffer via `memfd_create`, attaches to capture frame
- Pixel format conversion: ARGB8888 (BGRA in memory) → RGBA for PNG
- Auto-generates timestamped filenames in `~/Pictures/Screenshots/`
- Replaces previous `cosmic-screenshot` shell-out in Lua API

## cosmix-view header bar fix

Removed `core.window.header_title` from cosmix-view — it was filling the header
bar center area and blocking the system-level RMB context menu (Sticky, Minimize,
Move to Workspace, etc.) that COSMIC provides on all app title bars. cosmix-calc
and cosmix-mail already had the RMB menu working because they either don't set
the title or have enough empty header bar space.

## Lua library ports from nodemesh

Ported three modules from `~/.gh/nodemesh` Rust crates to Lua in `_lib/cosmix/`:

- **`amp.lua`** — AMP wire protocol parser/serializer (from `crates/amp/`).
  Markdown frontmatter format, validation, address parsing.
- **`email.lua`** — .eml file parser with HTML stripping (from
  `crates/docparse/extractors/email.rs`).
- **`mesh.lua`** — Route table, metrics counters, TOML config loader, dispatch
  logic (from `crates/meshd/`).
- **`dbus.lua`**, **`midi.lua`** — Additional Lua helper modules.

## GUI policy and libcosmic migration

- Added policy to `CLAUDE.md`: "All cosmix GUI apps must use libcosmic, not raw iced"
- **cosmix-calc**: Ported from raw `iced` to `cosmic::Application` trait. Uses
  `display_cache`/`memory_cache` fields to avoid MutexGuard lifetime issues in `view()`.

## Cosmic-edit style menu bars

Cloned the exact menu bar pattern from `cosmic-edit` source and applied across apps:

- **Pattern**: `responsive_menu_bar().item_height(Dynamic(40)).item_width(Uniform(280)).spacing(4.0)`
- **cosmix-view**: File/Edit/View/Help menus with keybinds (Ctrl+O open, Ctrl+S save,
  Ctrl+Q quit, Ctrl+=/- zoom, Left/Right prev/next). Removed old two-row toolbar.
- **cosmix-mail**: File/Edit/View/Help menus with keybinds (Ctrl+N compose, Ctrl+R reply,
  Ctrl+Q quit, F5 refresh, Del delete). Changed selection from `Button::Suggested`
  (rounded accent) to `Button::ListItem` (square highlight). Account switcher moved
  to `header_end()`.

## New crates

  for remote control (calc, result, clear, history, memory, press commands).
- **cosmix-mail**: Separated into own crate with `Cargo.toml`, `config.rs`, `tree_view.rs`.
