# 2026-04-03 — Fix cosmix-mcp tool registration (rmcp serve bug)

## Changes

- **Fixed MCP tools not loading in Claude Code** — root cause was `rmcp`'s `server.serve()` returns a `RunningService` handle that must be held alive. Without calling `service.waiting().await`, the handle was immediately dropped after the `initialize` handshake, killing the message loop before `tools/list` could be processed. Claude Code saw "connected" with instructions but zero callable tools.
- **Added tracing-subscriber** to cosmix-mcp for debugging rmcp internals via `RUST_LOG=trace`. This revealed the key log line: `RunningService dropped without explicit close()`.
- **All 13 MCP tools now register and work end-to-end:** amp_call, amp_list_services, amp_list_peers, hub_ping, log_tail, log_search, context_search, index_workspace, skills_retrieve, skills_store, skills_refine, skills_list, skills_delete.

## Notes

- The fix was a one-line change: `server.serve(stdio()).await` → `let service = server.serve(stdio()).await?; service.waiting().await`
